Domain Layer: Value Objects + Domain Exceptions + Aggregates
📖 THEORY
Domain-Driven Design fundamentals. Entity vs Value Object: Entity has identity (Product with Id), Value Object has no identity (Money, Address). Aggregate Root controls all access. Domain Events — things that happened (ProductCreated, PriceChanged) decouple side effects from business logic.
⚙️ PRACTICAL
Build in Domain: (1) ValueObjects/Money.cs: record Money(decimal Amount, string Currency) with validation. (2) Exceptions/DomainException.cs + ProductExceptions.cs. (3) Events/ProductCreatedEvent.cs. (4) Update Product: private setters, constructor validates (name not empty, price not negative), method DecreaseStock(int qty) throws InsufficientStockException.
⚡ KEY INSIGHT
Private setters on entity properties enforce that all changes go through methods. Entity constructors validate invariants — if you have an entity in memory, it is ALWAYS valid.
🎯 DELIVERABLE
Product entity with validation, Money value object, domain exceptions. Zero external dependencies. Push.

Application Layer: Interfaces + DTOs + Use Case Structure
📖 THEORY
Application layer defines WHAT the system does (use cases), not HOW. It depends on interfaces (defined in Application), not concrete implementations (in Infrastructure). IProductRepository lives in Application — Infrastructure implements it. This is Dependency Inversion Principle in action.

Application Layer: MediatR + First Use Case (GetProductById)
📖 THEORY
CQRS — separate reading (queries) from writing (commands). Commands change state. Queries read state, never change it. MediatR: Controller sends Request → MediatR finds Handler → Handler executes → returns Response. Benefit: controllers become thin. All business logic in handlers — testable without HTTP context.
⚙️ PRACTICAL
dotnet add package MediatR. Create: (1) GetProductByIdQuery.cs: record GetProductByIdQuery(Guid Id) : IRequest. (2) GetProductByIdQueryHandler.cs: implements IRequestHandler, constructor takes IUnitOfWork, Handle method fetches and maps to DTO. (3) ProductMappingExtensions.cs: static ToDto() extension.
💻 CODE TASK
public class GetProductByIdQueryHandler : IRequestHandler { private readonly IUnitOfWork _uow; public GetProductByIdQueryHandler(IUnitOfWork uow) => _uow = uow; public async Task Handle(GetProductByIdQuery req, CancellationToken ct) { var product = await _uow.Products.GetByIdAsync(req.Id, ct); return product?.ToDto(); } }
⚡ KEY INSIGHT
Handlers depend only on IUnitOfWork (an interface). You can unit-test this handler with a mock IUnitOfWork — no HTTP, no DB needed. That is the entire point of Clean Architecture.
🎯 DELIVERABLE
GetProductById query + handler working. Unit test: mock IUnitOfWork, verify handler returns null for non-existent Id. Push.

Application Layer: CreateProduct Command + Result Pattern
📖 THEORY
Result pattern — instead of throwing exceptions for expected failures, return a Result object that carries either a success value or an error. Exceptions are expensive (stack unwind). Result makes error handling explicit and forces callers to handle failures.
⚙️ PRACTICAL
Build: (1) Common/Result.cs: generic with Success(T value) and Failure(string error) factory methods. (2) CreateProductCommand.cs: record with Name, Description, Price, Stock. (3) CreateProductCommandHandler.cs: validate inputs (return Failure if invalid), create Product entity, add to repo, save, return Success(product.Id).
💻 CODE TASK
var product = new Product(Guid.NewGuid(), command.Name, command.Description, new Money(command.Price, "INR"), command.Stock); await _uow.Products.AddAsync(product, ct); await _uow.SaveChangesAsync(ct); return Result.Success(product.Id);
🎯 DELIVERABLE
CreateProduct command + handler + Result pattern. Unit test: verify handler returns Failure when name is empty. Push.

DI lifetime differences?
Transient=new per injection. Scoped=new per request. Singleton=one per app. Captive dependency bug: Singleton depending on Scoped — fix with IServiceScopeFactory.
ConfigureAwait(false)?
Tells awaiter not to capture SynchronisationContext. Use in library code. Prevents deadlocks where SyncCtx is single-threaded. Not needed in ASP.NET Core but good practice.
IEnumerable vs IQueryable?
IEnumerable: in-memory LINQ. IQueryable: expression tree → SQL. Using IEnumerable on DbSet fetches all rows first — catastrophic performance.
What is CQRS?
Separate read (Query) and write (Command) paths. Commands change state. Queries read state, never change it. Handlers are focused, testable, independently scalable.
What is Clean Architecture?
4 layers: Domain (no dependencies), Application (→Domain), Infrastructure (implements Application interfaces), API (→Application). Dependency rule: always points inward.
Cache-aside pattern?
Check cache → hit: return. Miss: query DB → write to cache with TTL → return. Invalidate cache on writes.
Outbox pattern?
Write event to outbox table in same DB transaction as business data. Background worker reads outbox and publishes to broker. Guarantees at-least-once delivery.
Circuit breaker states?
Closed: requests flow. Open: fail fast (no calls). Half-open: one test request. 5 failures → Open. Success in Half-open → Closed.
RabbitMQ over Kafka?
RabbitMQ: push-based, flexible routing, easier ops. Kafka: pull-based, log-based, replay events, millions/sec. Kafka for streaming at scale, RabbitMQ for task/command messaging.
Domain vs Integration Events?
Domain: within bounded context, synchronous (same transaction). Integration: cross-service, asynchronous (via message broker). Domain events can trigger integration events.
Optimistic vs Pessimistic concurrency?
Optimistic: fail if another writer changed data (rowversion). No lock held. Correct for web APIs. Pessimistic: lock row before reading. Only for very short operations.
IAM Instance Profile?
Role attached to EC2. EC2 calls AWS services using IMDS temporary credentials. No access keys stored. THE correct pattern for EC2→AWS auth.
ECS Fargate vs EC2?
Fargate: serverless — AWS manages host. No patching, no cluster management. Pay per task CPU/memory second. EC2: you manage EC2 instances — can use Reserved Instances for cost.
Why RDS in private subnet?
DB should not be reachable from internet. SG allows port 5432 only from application SG. If DB is in public subnet, it is directly attackable.
SOLID — one example each?
S: Controller handles HTTP only. O: Pipeline behaviour adds validation without modifying handler. L: Any IProductRepository impl is substitutable. I: IProductReadRepository vs IProductWriteRepository. D: Application depends on interface, not EfRepository.
Presigned URL — what and why?
Time-limited signed URL for one specific S3 operation. Client uploads/downloads directly to S3 — no API proxying. Secure (no public bucket), scalable (no server bandwidth).
N+1 problem + fix?
Load 100 products (1 query), then 100 queries for their categories = 101 total. Fix: .Include(p=>p.Category) generates single JOIN. For multiple collections: .AsSplitQuery().
Distributed lock — how?
Redis: SET key token NX PX ttl_ms (atomic). Set only if not exists (NX). Release only if your token (Lua script for atomicity). Prevents concurrent cache rebuild or job double-execution.
Rate limiter — sliding window counter?
count = current_window + (previous_window × (1 - elapsed/window_size)). O(1) space, more accurate than fixed window. Prevents boundary burst that fixed window allows.
Microservice boundaries?
Split by business capability, not technical layer. Each owns its data. Communicate via events or APIs (no shared DB). Start coarse-grained, split when teams diverge.
ALB vs NLB?
ALB (Layer 7): HTTP/HTTPS, route by URL/host, SSL termination, WAF. NLB (Layer 4): TCP/UDP, ultra-low latency, static IPs, millions req/sec. Use ALB for web APIs.
Saga pattern?
Manage distributed transactions without 2PC. Choreography: services react to events. Orchestration: central coordinator. Compensating transactions for rollback.
Strategy pattern usage?
PricingStrategy: Standard, BulkDiscount, Seasonal all implement IPricingStrategy. Factory selects at runtime. Adding new pricing rule does not modify existing strategies.
Decorator over inheritance for caching?
Decorator wraps same interface, composable at runtime. CachingRepository(EfRepository) OR CachingRepository(InMemoryRepository) — interchangeable. Inheritance is static, tightly coupled.
CI/CD pipeline — your setup?
Push to main → GitHub Actions CI (build+test+coverage) → docker build + push to ECR (:latest + :git-sha) → deploy to ECS Fargate (update task definition + rolling update) → CloudWatch health check. Total: < 8 minutes.
How do you handle secrets?
AWS Parameter Store SecureString (KMS-encrypted). ECS reads via IAM role (no access keys). No secrets in code, .env files, or baked into images.
URL Shortener — scale?
10:1 read:write. 11,600 reads/sec. Cache top 20% in Redis. Read replicas for DB. Counter-based short code generation (no collision). CDN for popular redirects.
What is Outbox Pattern vs direct publish?
Direct: save to DB, publish to broker — crash between = event lost. Outbox: save event to outbox table in same DB transaction. Worker publishes. Guaranteed delivery.
Good questions to ask interviewer?

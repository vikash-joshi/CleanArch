Daily 2-hour plan — non-negotiable structure
Split into two 60-min blocks. Block A = learning. Block B = building. Never do both in the same hour.
0:00 – 0:05
Review yesterday's notes (5 min)
Not optional. Spaced repetition is free recall gains.
0:05 – 1:00
Block A — Learn (55 min)
Month 1: DSA problem (25 min) + .NET/SQL concept (30 min). Month 2: Docker/microservices tutorial + implement one piece. Month 3: AWS console hands-on or GitHub Actions config. Month 4: System design video + hand-draw the diagram.
1:00 – 1:55
Block B — Build (55 min)
Always coding. Never watching tutorials. If you can't write it from memory, you haven't learned it. Push to GitHub every session.
1:55 – 2:00
Log what you built + one thing you'll do tomorrow
Accountability. Also gives you material for interview STAR answers.
Weekend rules — don't skip these
Saturday (3 hrs): Build a meaningful chunk of the production project. No tutorials. Pure building.

Sunday (2 hrs): 1 LeetCode medium + 1 system design review (or mock interview in Month 4). Then rest. Burnout at Month 2 kills the entire plan.

Hard rule One full day completely off per week. No coding, no YouTube. Your brain consolidates during rest.

What interviewers will ask about this project
"Why microservices over monolith?" — Answer: team scalability, independent deployments, tech freedom per service. Be honest that for this scale a monolith was also fine — that nuance impresses seniors.

"How do services communicate?" — REST for sync (order status), RabbitMQ for async (stock updates, email triggers). Know when you chose which and why.

"What happens if RabbitMQ goes down?" — Retry with exponential backoff + dead-letter queue. If you've actually implemented this, say so.

"How does your CI/CD work?" — GitHub push → Actions builds image → pushes to ECR → SSH to EC2 → docker pull + restart. Walk them through it step by step.

Résumé bullet formula — stop listing responsibilities
Wrong "Developed REST APIs using ASP.NET Core"
Right "Built 12 REST APIs serving 50k daily requests, reduced avg response time from 800ms to 180ms by introducing Redis cache layer"

Wrong "Worked on SQL Server database optimization"
Right "Optimised 3 critical reports by rewriting N+1 queries and adding composite indexes, cutting load time from 14s to 1.2s"

Every bullet: Action + What + Metric. If you don't have a metric, estimate one honestly.
Salary negotiation — don't leave 2–4 LPA on the table
Never disclose your current 6.6 LPA first. When asked, say "I'm targeting 14–16 LPA based on market rates for this role and my background." That anchors the negotiation high.

Always negotiate after the verbal offer. Say: "I'm excited about this role. I did have one other discussion at 15 LPA — is there flexibility?" Even if the other offer is informal, this is fair to say.

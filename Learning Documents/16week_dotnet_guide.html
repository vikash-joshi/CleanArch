Your daily 2-hour structure — follow this every single day
0:00 – 0:05 → Recall: from memory, recall what you built yesterday (no peeking)
0:05 – 1:00 → Theory block: read, study, understand the concept
1:00 – 1:55 → Practical block: type code, run it, break it, fix it
1:55 – 2:00 → Log: 1-sentence commit message AND push to GitHub
Saturday → 3-hour extended build: work on the weekly deliverable
Sunday → 1-hour review only: no new topics, no tutorials, just revisit
Non-negotiable rules
1. NEVER copy-paste code you did not understand line by line
2. Push to GitHub EVERY session — no exceptions, even 5-line changes
3. If you miss a day: continue from where you stopped, do NOT double up
4. Start applying from Week 11. Do NOT wait until you feel ready
5. Build depth, not breadth. Know 5 things cold vs 15 things vaguely
